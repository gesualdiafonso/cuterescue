API Geolocalización de Pets

Gestión de Pets, Dueños y Localización, por ahora solo se maneja en tiempo 0 absoluto cuando creas el pet. AL futuro se manejará en tiempo real.

La Cute Rescue API permite gestionar la información de mascotas y dueños, así como la geolocalización de los animales en tiempo real. Toda la información está almacenada en MongoDB Atlas y los endpoints cuentan con validaciones de datos. por ahora solo se maneja en tiempo 0 absoluto cuando creas el pet. Al futuro se manejará en tiempo real.

¿Qué hace esta API?

Esta API REST provee operaciones CRUD para:

  • Usuarios: Creación de usuario (email y contraseña)
  • Dueños: Creación de detalles adicionales del usuarios como: (nombre, teléfono, ubicación).
  • Pets: Registro de las pets, actualización, búsqueda y eliminación de mascotas, con identificación por chip_id.
  • Localización: Creación automática en la posición inicial (0), registro de nuevas coordenadas y consulta de histórico.
⚡ Recordamos que por ahora solo se maneja en tiempo 0 absoluto cuando creas el pet. Al futuro se manejará en tiempo real. Y cada data tiene su propio id

Endpoints de la API

Los endpoints son las rutas que permiten interactuar con la API. A través de ellos se pueden realizar operaciones GET, POST, PUT y DELETE sobre los recursos de Pets, Dueños y Localización.

  • Rota /user: Permite la creación de usuarios y la gestión de sus datos (email y contraseña)
  • Rota /user/:userId/details: Permite la creación de usuarios y la gestión de sus datos (nombre, email, teléfono, ubicación, etc.).
  • Rota /pets: Permite registrar, consultar, actualizar y eliminar mascotas usando su chip_id.
  • Rota /locations: consultar el historial de movimientos de cada mascota.

Cómo acceder a los Endpoints

Cada endpoint es una ruta específica para interactuar con un recurso. Los métodos soportados son GET, POST, PUT y DELETE.

Observación: Los campos de Id creado en el usuarios son unicos y van a reflejar en comunicación de los demás campos como: userId, dueno_id, estes campos son obligatorios y de recuerdo dado que son campos hasheados en hex. No comunicamos nuestros id con _id.

  • GET /api/users → Listar todos los usuarios
  • GET /api/users/:id → Consultar usuario por ID
  • POST /api/users → Crear usuario (email y contraseña)
    • 📌 Ejemplo de respuesta (GET/POST) 
    • {
                        "id": "hash-id",
                        "email": "string",
                        "password": true,
                        "activo": true,
                        "pets": []
                      }
  • POST /api/users/details → Agregar detalles del dueño
  • PUT /api/users/:id → Actualizar datos del dueño
  • DELETE /api/users/:id → Eliminar dueño

  • GET /api/users → Listar todos los detalles de los usuarios creado
  • GET /api/users/:userId/details → Consultar detalles según el Id
  • POST /api/users/:userId/details → Crear los detalles de los usuario (llevar userId (id), nombre, telefono, ubicacion pets), en pets de inicio es vacio
    • 📌 Ejemplo de respuesta (GET/POST) 
    •                           
                            {
                            "userId": "stringId",
                            "nombre": "string(required)",
                            "telefono": "string(required)",
                            "ubicacion": "string(required|)",
                            "pets": []
                          }
  • PUT /api/users/:userId/details → Actualizar detalles del usuario
  • DELETE /api/users/:userId/details → Elimina los detalles del usuario, Aviso: No borres el detalle sin antes borrar al pet que está asignado a lo usuario!

  • GET /api/pets → Listar todas las mascotas existente en la base de datos
  • GET /api/pets/:id → Obtener mascota por ID
  • POST /api/pets → Crear nueva mascota (Todas las propiedades son obligatorias) y en Aviso: en el campo de ultima_localizacion se debe dejar vacio que se creeará automaticamente al crear la mascota. Observación Importante: en campo dueno_id se debe traer el id del userId creado, si no lo recuerdas puedes volver al GET /api/user para buscar el id del ultimo usuario creado
    • 📌 Ejemplo de respuesta (GET/POST) 
    •                         {
                          "id": "hash-id",
                          "chip_id": "CHIP-HASH",
                          "nombre": "string",
                          "especie": "string",
                          "raza": "string",
                          "dueno_id": "hash-id",
                          "fecha_nacimiento": "string",
                          "edad": number,
                          "sexo": "string",
                          "color": "string",
                          "foto_url": "string",
                          "activo": true,
                          "ultima_localizacion": {object},
                          "estado_salud": "string",
                          "alertas": [],
                          "collar_bateria": number,
                          "nivel_actividad": "string"
                        }
  • GET /api/pets/:chip_id → Va poder buscar la mascota según el campo chip_id generado del POST
  • PUT /api/pets/:id → Actualizar datos de mascota
  • DELETE /api/pets/:id → Eliminar mascota, que por defecto elemina del detalles del usuario

  • GET /api/locations → Busca a todas las ultimas ubicación creadas en el punto 0
  • GET /api/locations/:chip_id → Consultar histórico por chip_id de la mascota

⚡ Al crear un Pet, se genera automáticamente una localización inicial en la posición 0.

Presentación del Proyecto Final

El proyecto Cute Rescue se plantea como una solución integral para la geolocalización y seguridad de mascotas. La propuesta final combina:

  • Aplicación Nativa: Construida con frameworks híbridos para permitir el acceso multiplataforma (Android/iOS).
  • ESP32: Microcontrolador que captura la ubicación GPS del pet y envía la información hacia la API.
  • API REST: Punto centralizado de comunicación, validado y conectado con MongoDB Atlas para almacenar mascotas, dueños y localizaciones.

El ecosistema completo demuestra cómo integrar hardware, software y servicios digitales para dar soporte a un escenario real de cuidado animal.
